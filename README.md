#  Revenue Analytics Dashboard

## Project Overview

In this project, I developed an interactive revenue analytics dashboard to analyze cash flow dynamics and understand the key drivers behind revenue changes.

The main objective was to help product managers monitor monthly performance, identify growth opportunities, and detect potential revenue risks through data-driven insights.

The solution combines SQL-based data extraction with interactive data visualization.

---

##  Tools & Technologies

* **SQL (PostgreSQL)** – data extraction and transformation
* **Tableau Public** – dashboard design and visualization
* **Data Analysis & Visualization** techniques

---

##  Data Source

Data was retrieved from a PostgreSQL database (`project` schema) by writing custom SQL queries to calculate all key metrics.

## Key Metrics Tracked

The dashboard provides a comprehensive view of revenue performance using the following metrics:

* Monthly Recurring Revenue (MRR)
* Paid Users
* Average Revenue Per Paid User (ARPPU)
* New Paid Users & New MRR
* Churned Users & Churn Rate
* Churned Revenue & Revenue Churn Rate
* Expansion MRR & Contraction MRR

---

## Key Insights

* Revenue growth is largely driven by **new paid users and expansion revenue**.
* **Churn has a direct negative impact** on overall revenue stability.
* ARPPU remains relatively stable, indicating **consistent user spending behavior**.
* Periods of revenue decline are strongly linked to **increases in churn rate**.

## Dashboard Features

* Interactive and user-friendly design
* Time-based trend analysis (monthly)
* Filters for:

  * Date
  * User language
  * User age
